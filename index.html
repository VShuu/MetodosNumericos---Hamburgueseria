01 — Planteamiento

El Problema Real

Una hamburguesería debe decidir cuántas unidades de cada ingrediente principal preparar cada día para satisfacer la demanda, respetar el inventario y maximizar la eficiencia.

Modelamos esto como un sistema de ecuaciones lineales Ax = b, donde el equilibrio entre recursos, producción y demanda queda capturado algebraicamente.

Variables del sistema

x₁
Carne (kg) Cantidad de carne preparada por turno
x₂
Pan (unidades) Panes frescos disponibles por turno
x₃
Ingredientes extra Queso, salsas y vegetales (porciones)
b
Términos independientes Restricciones de demanda, presupuesto y capacidad de almacenamiento

Los tres escenarios

✅

Caso Ideal

Sistema consistente y bien condicionado. Coeficientes equilibrados representan una demanda normal. Todos los métodos convergen rápidamente.

🔥

Caso Bajo Estrés

Alta demanda: coeficientes grandes. Simula un evento especial o fin de semana. Pone a prueba la estabilidad de los métodos iterativos.

⚠️

Caso Mal Condicionado

Ingredientes casi equivalentes generan problemas numéricos. El número de condición es muy alto. Ideal para demostrar el valor del precondicionamiento.

02 — Fundamentos

Métodos Numéricos

Factorización LU

Descompone la matriz A en el producto de una matriz triangular inferior L y una triangular superior U, tal que A = LU. Luego resuelve dos sistemas triangulares:

  • Ly = b (sustitución hacia adelante)
  • Ux = y (sustitución hacia atrás)

En el contexto de la hamburguesería: una vez factorizada A, podemos resolver para múltiples vectores b (distintos días) de forma eficiente.

O(n³) factorización · O(n²) por solución
ALGORITMO LU (con pivoteo parcial)
Para k = 0 a n-1:
  Encontrar fila pivot (max |A[i][k]|, i≥k)
  Intercambiar filas k y pivot
  Para i = k+1 a n-1:
    m = A[i][k] / A[k][k]   ← multiplicador
    L[i][k] = m
    Para j = k a n-1:
      A[i][j] -= m * A[k][j]
Resolver Ly = Pb (sustitución ↓)
Resolver Ux = y  (sustitución ↑)

Método de Jacobi

Método iterativo que actualiza todas las variables simultáneamente usando los valores de la iteración anterior:

x(k+1)i = (bi − Σj≠i aij x(k)j) / aii

Requiere diagonal dominante para garantizar convergencia. En la hamburguesería: cada ingrediente se recalcula independientemente.

Convergencia: O(ρ(D⁻¹(L+U))^k)
ALGORITMO JACOBI
x = x0  (vector inicial, ej: ceros)
Para iter = 1 a MAX_ITER:
  Para i = 0 a n-1:
    suma = 0
    Para j = 0 a n-1:
      Si j ≠ i: suma += A[i][j] * x[j]
    x_nuevo[i] = (b[i] - suma) / A[i][i]
  error = ||x_nuevo - x|| / ||x_nuevo||
  x = x_nuevo
  Si error < tol: CONVERGIÓ ✓

Gauss-Seidel

Mejora sobre Jacobi: usa los valores ya actualizados en la misma iteración. Converge aproximadamente el doble de rápido:

x(k+1)i = (bi − Σj<i aij x(k+1)j − Σj>i aij x(k)j) / aii

En la hamburguesería: al ajustar un ingrediente, inmediatamente influye en el cálculo del siguiente.

~2× más rápido que Jacobi
ALGORITMO GAUSS-SEIDEL
x = x0
Para iter = 1 a MAX_ITER:
  Para i = 0 a n-1:
    suma = 0
    Para j = 0 a n-1:
      Si j ≠ i: suma += A[i][j] * x[j]
      // Usa x[j] ya actualizado si j < i
    x[i] = (b[i] - suma) / A[i][i]
  error = ||Ax - b|| / ||b||
  Si error < tol: CONVERGIÓ ✓

SOR — Relajación Sucesiva

Generaliza Gauss-Seidel con un factor de relajación ω ∈ (0, 2):

x(k+1)i = (1−ω)x(k)i + ω·xGSi

  • ω = 1: equivale a Gauss-Seidel
  • ω < 1: sub-relajación (estabiliza)
  • ω > 1: sobre-relajación (acelera)

Para la hamburguesería bajo estrés, ajustar ω puede reducir drásticamente las iteraciones.

ω óptimo ≈ 2/(1+√(1-ρ²))
ALGORITMO SOR (ω parametrizable)
x = x0
Para iter = 1 a MAX_ITER:
  Para i = 0 a n-1:
    sigma = 0
    Para j = 0 a n-1:
      Si j ≠ i: sigma += A[i][j] * x[j]
    x_gs = (b[i] - sigma) / A[i][i]
    x[i] = (1 - ω) * x[i] + ω * x_gs
  error = ||Ax - b|| / ||b||
  Si error < tol: CONVERGIÓ ✓

Gradiente Conjugado Precondicionado

El método del gradiente conjugado resuelve Ax = b minimizando la función cuadrática f(x) = ½xᵀAx − bᵀx siguiendo direcciones de búsqueda conjugadas (A-ortogonales).

El precondicionador M transforma el sistema a M⁻¹Ax = M⁻¹b, reduciendo el número de condición y acelerando la convergencia dramáticamente para el caso mal condicionado.

Converge en ≤ n iteraciones (exacto) · O(κ^½) con precond.
ALGORITMO PCG (Precondicionado)
x = x0; r = b - A*x
z = M⁻¹ * r  (aplicar precondicionador)
p = z; rz = r·z
Para iter = 1 a MAX_ITER:
  Ap = A * p
  α = rz / (pᵀ Ap)    ← paso óptimo
  x = x + α*p
  r = r - α*Ap
  Si ||r|| < tol: CONVERGIÓ ✓
  z = M⁻¹ * r
  rz_nuevo = r·z
  β = rz_nuevo / rz    ← conjugación
  p = z + β*p
  rz = rz_nuevo
03 — Optimización

El Enfoque del Gradiente

Función a minimizar f(x) = ½ xᵀAx − bᵀx + c
Condición de óptimo ∇f(x) = Ax − b = 0
Equivalente a resolver Ax = b ✓

Interpretación hamburguesera: el gradiente ∇f(x) indica en qué dirección y cuánto ajustar los ingredientes para reducir el "error" entre la producción actual y la demanda óptima. Cuando ∇f = 0, hemos encontrado el punto de equilibrio perfecto.

Número de condición κ(A)

κ(A) = ‖A‖ · ‖A⁻¹‖ = λmax/λmin

  • κ ≈ 1: bien condicionado, convergencia rápida
  • κ ≫ 1: mal condicionado, convergencia lenta o inestable
  • Precondicionador: reduce κ(M⁻¹A) ≪ κ(A)

🍔 Interpretación en la Hamburguesería

✅

Día Normal (Caso Ideal)

Todos los métodos funcionan bien. LU es la elección óptima por su exactitud. Gauss-Seidel converge en pocas iteraciones. La hamburguesería opera con fluidez.

🔥

Evento Especial (Caso Estrés)

Con alta demanda, SOR con ω óptimo supera a Jacobi/GS. El gradiente conjugado brilla aquí. Necesitamos algoritmos robustos para no quedarnos sin ingredientes.

⚠️

Ingredientes Similares (Mal Cond.)

Jacobi y GS pueden divergir o tardar miles de iteraciones. El PCG con precondicionador Jacobi reduce κ y converge. LU con pivoteo sigue siendo confiable.

📈

Conclusión General

Para una hamburguesería real: usar LU para planificación diaria, PCG para optimización en tiempo real con muchos ingredientes (n grande), y SOR cuando se requiere velocidad iterativa.
